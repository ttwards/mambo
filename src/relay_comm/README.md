# relay_comm — 机械臂区域控制板

下位机，运行在 STM32 控制板上，负责 3 路继电器控制、CAN 电机控制和机械臂抓取轨迹执行。

## 硬件

| 功能 | 引脚/总线 |
|------|-----------|
| 继电器 1 | PE9 |
| 继电器 2 | PE11 |
| 继电器 3 | PE13 |
| 通信 UART | usart6 |
| 电机 CAN | can1 |

### 电机

| 电机 | CAN ID | 类型 | 角色 |
|------|--------|------|------|
| rs_motor1 | 0x01 | RS 电机 | 预留 |
| rs_motor2 | 0x02 | RS 电机 | 大臂 (肩关节) |
| rs_motor3 | 0x03 | RS 电机 | 小臂 (肘关节) |
| dm_motor | 0x04 | DM 电机 | 腕部旋转 (吸嘴 theta) |

## 协议

通过 ARES UART 双协议通信，100Hz 帧率。

### 接收帧 (0x0200, 上位机→本板) 8 float

| 索引 | 含义 |
|------|------|
| [0] | 模式选择: 1=模式1/2, 2=模式3 |
| [1] | 子模式: 200=模式1(正前朝前吸), -200=模式2(正前朝下吸) |
| [2..7] | 保留 |

### 发送帧 (0x0100, 本板→上位机) 8 float

| 索引 | 含义 |
|------|------|
| [0] | 动作完成标志: DONE(0x4B0CD0F1) / PENDING(0x4E0DD0F1) |
| [1] | motor2 反馈角度 |
| [2] | motor3 反馈角度 |
| [3] | dm_motor 反馈角度 |
| [4] | 保留 |
| [5..7] | 3 路继电器电平 (1.0=高, 0.0=低) |

## 三种抓取模式

| rx_data[0] | rx_data[1] | 模式 | 说明 | 完成后继电器1 |
|------------|------------|------|------|--------------|
| 1 | 200 | 模式1 | 正前朝前吸 | 打开 (PE9=HIGH) |
| 1 | -200 | 模式2 | 正前朝下吸 | 打开 (PE9=HIGH) |
| 2 | 任意 | 模式3 | 背后朝前吸 | 关闭 (PE9=LOW) |

三种模式均使用三次贝塞尔曲线插值 + 2-DOF 逆运动学求解，共 20 步，每步间隔 700ms，全程约 14 秒。

**重要**: 上位机指令不直接控制继电器。继电器由抓取模式完成后自动控制。

## 脉冲机制

- 收到指令 → 执行抓取轨迹 → 发送 10 帧 DONE 标志 (100ms @ 100Hz) → 自动回到 PENDING
- 脉冲期间屏蔽新指令，防止完成标志重叠导致误触发
- DONE/PENDING 使用 uint32 magic number 映射为 float，避免串口噪声误判

## 编译

```bash
west build -b robomaster_board_c src/relay_comm
```

## 测试

通过串口发送测试指令：

```c
// 触发模式1 (正前朝前吸)，完成后 PE9=HIGH
float cmd1[] = {1.0f, 200.0f, 0, 0, 0, 0, 0, 0};

// 触发模式2 (正前朝下吸)，完成后 PE9=HIGH
float cmd2[] = {1.0f, -200.0f, 0, 0, 0, 0, 0, 0};

// 触发模式3 (背后朝前吸)，完成后 PE9=LOW
float cmd3[] = {2.0f, 0, 0, 0, 0, 0, 0, 0};
```
